PURPOSE: Prompt template for explaining what a module/file does

Content structure:
- System role definition (senior software engineer analyzing code)
- Task description (analyze this module and explain its purpose)
- Required output format:
  * 2-3 sentence summary
  * List of key responsibilities
  * How it fits into larger system
  * Notable dependencies
- Constraints (be concise, avoid jargon, focus on intent not implementation)
- Example input/output pair for few-shot learning

Variables to template in:
- {file_name}
- {file_content}
- {imports}
- {context} (surrounding modules)
