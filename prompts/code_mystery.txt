PURPOSE: Prompt template for explaining unusual/complex code

Content structure:
- System role (experienced developer debugging/reviewing code)
- Task (explain why this code exists and what problem it solves)
- Context (this code is unusual because: high complexity, weird naming, etc.)
- Required output format:
  * What the code does
  * Likely problem it solves
  * Why it might be written this way
  * Potential issues or improvements
- Constraints (be hypothesis-driven, acknowledge uncertainty)

Variables to template in:
- {mystery_code}
- {mystery_reason} (why flagged as mystery)
- {surrounding_context}
